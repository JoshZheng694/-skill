# Study Guide Generator

将教材PDF/OCR分块 + 复习指南模板 → 完整背诵资料的端到端闭环流水线。

## 触发条件

Use this skill when the user says any of:
- "创建复习资料" / "做复习提纲" / "整合考点"
- "/study-guide-generator"
- "把教材内容填入提纲" / "复习资料审阅补充"
- References a textbook with an outline that needs filling

## 前提

必须已有：
1. **教材源文件**：PDF或OCR后MD分块（如 `原始文件/chunk_*.md`）
2. **复习指南模板**：包含考点列表+页码索引的提纲（如 `.tmp/经济史复习指南.md`）

如果没有教材OCR分块，先使用 `/pdf-to-markdown` 技能进行OCR转换。

## 完整闭环流水线

```
教材OCR分块(MD) + 复习指南模板
         │
         ▼
┌─────────────────────────────────────┐
│ 阶段一：提取 (10代理并行)            │
│ • 主代理写plan.md + 分工             │
│ • 每个代理分配~4-5个chunk + 全模板   │
│ • 逐考点搜索教材原文，填入模板        │
│ • 每个代理立即保存partial结果到本地   │
└─────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│ 阶段二：审阅 (10代理并行)            │
│ • 每个审阅代理负责1-2个板块           │
│ • 逐考点核对教材原文准确性            │
│ • 替换所有"……"/"..."为完整原文       │
│ • 补充遗漏考点，修正页码              │
│ • 区分"教材原文"vs"课堂补充"          │
└─────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│ 阶段三：元审阅 (2代理并行)           │
│ • R-A: 交叉检查板块一~四完整性       │
│ • R-B: 交叉检查板块五~八+全局一致性  │
│ • 产出问题清单 + 修复建议             │
└─────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│ 阶段四：整合交付 (1教师代理)         │
│ • 合并所有审阅修正到最终文档          │
│ • 统一页码/格式/标注                  │
│ • 产出CHANGELOG + 最终复习资料        │
└─────────────────────────────────────┘
```

## 阶段一：提取规划

主代理使用 `/do-agent` 模式。关键约定：

### 1.1 工作目录
```
agent_tasks/{task_name}_YYYYMMDDHH/
├── plan.md              ← 第一份输出：执行计划
├── 提纲模板.md           ← 复习指南副本（供子代理填充）
├── partial_A{N}.md      ← 子代理输出
└── 最终输出.md
```

### 1.2 分块分配方案（10个提取代理）

| Agent | 分块 | 页码范围 |
|-------|------|----------|
| A1 | chunk_01~04 | p1-40 |
| A2 | chunk_05~08 | p41-80 |
| A3 | chunk_09~12 | p81-120 |
| A4 | chunk_13~16 | p121-160 |
| A5 | chunk_17~20 | p161-200 |
| A6 | chunk_21~24 | p201-240 |
| A7 | chunk_25~28 | p241-280 |
| A8 | chunk_29~32 | p281-320 |
| A9 | chunk_33~36 | p321-360 |
| A10 | chunk_37~42 | p361-419 |

### 1.3 提取代理工作规范

每个提取代理：
1. **自己读取**分配的chunk文件 + 完整复习指南模板
2. 对照复习指南的每个考点，在chunk中搜索相关内容
3. 提取格式：
   ```
   ### N. 考点名称 【优先级：★必考/重点/了解】
   **教材原文**（第X页）："完整原文引用"
   **考试要点**：……
   **答题关键词**：……
   ```
4. 如果考点在分配范围内未找到，标注"（未覆盖）"
5. **立即保存**结果到指定输出文件，不返回主代理
6. 返回状态摘要：找到几个考点、未覆盖几个

## 阶段二：审阅规划

主代理启动10个审阅代理并行执行。关键分工：

### 2.1 审阅分工表

| 代理 | 负责板块 | 教材分块 | 重点任务 |
|------|---------|---------|---------|
| R01 | 板块一前半 | chunk_01~04 | 修复省略号，修正页码偏移 |
| R02 | 板块一后半 | chunk_03~08 | 填补遗漏考点，验证课堂补充 |
| R03 | 板块二前半 | chunk_07~12 | 验证赋税制度演变链 |
| R04 | 板块二后半 | chunk_05~12 | 找遗漏考点教材出处 |
| R05 | 板块三 | chunk_07~14 | 验证商业/货币/丝绸之路 |
| R06 | 板块四前半 | chunk_13~18 | 填补真空区+验证因果链 |
| R07 | 板块四后半+五开头 | chunk_17~24 | 近代经济形态+当代起点 |
| R08 | 板块五 | chunk_21~32 | 补全A8缺失的教材原文 |
| R09 | 板块六+七 | chunk_31~42+导论 | 搜索理论来源+新发展内容 |
| R10 | 板块八+全局 | chunk_37~42+全范围 | 前沿议题+省略号全盘扫描 |

### 2.2 审阅质量标准

核心规则（必须编码到每个审阅代理提示词中）：

**省略号零容忍**：
- 严禁出现 `"……"`、`"..."`、`"等等。"` 结尾的教材引文
- 必须完整摘录教材句子，或使用 `[中间N段略]` 明确标记跳过部分
- 对于过长的段落（>300字），使用两条或多条完整引文

**页码统一**：
- 统一使用教材印刷页码（即教材页脚标注的页码）
- 如果OCR分块页码与教材印刷页码存在偏移，以教材印刷页码为准
- 消除所有"对应教材第X页"的过渡性标注

**课堂vs教材区分**：
- 教材有直接论述的：标注页码 + 完整原文
- 教材无直接论述的：标注 `> ⚠️ 课堂补充：教材未设专门章节，以下为教师课堂讲授内容`
- 教材有间接素材的：标注 `> ⚠️ 部分课堂补充：教材提供相关实证素材（见第X-Y页），理论框架为课堂讲授`

**逐考点验证**：
- 每条"教材原文"引用必须能在对应的OCR分块中找到原文
- 如果原文与提纲标注的页码不符，以实际找到的页码为准并修正
- 如果一个考点被多个代理覆盖，取教材原文最完整、页码最准确的版本

### 2.3 审阅代理输出格式

每个审阅代理输出完整的审阅后板块内容到：
`agent_tasks/{task_name}_YYYYMMDDHH/review_R{N}_{板块}.md`

每个考点输出格式：
```
### N. 考点名称 【优先级：★必考/重点/了解】

**教材原文**（第X页）："完整原文引用，不使用省略号"

**考试要点**：……

**答题关键词**：……
```

## 阶段三：元审阅

启动2个元审阅代理交叉检查全部10个审阅输出：

| 代理 | 范围 | 重点 |
|------|------|------|
| R-A | 板块一~四 | 覆盖完整性、审阅间矛盾、省略号修复核实、关键缺口状态 |
| R-B | 板块五~八+考前Checklist | R08深度验证（抽查原文准确性）、R09理论搜索验证、全局一致性 |

元审阅输出：
- 覆盖完整性表（每个考点状态）
- 审阅间矛盾发现
- 省略号修复抽查结果
- 关键缺口状态（已解决/仍需修复）
- 整体就绪判定（READY / NEEDS-FIX on N items）

## 阶段四：整合交付

启动1个教师代理完成最终整合。任务：

1. **读取全部输入**：10个审阅输出 + 2个元审阅报告 + 当前完整提纲
2. **应用所有修正**：页码、省略号、内容补充、课堂标记
3. **统一格式**：
   - 考试题型表 → 一~八板块 → 考前Checklist → 答题套路
   - 每个考点统一 `### N. 考点名称 【优先级】` 格式
   - 教材原文统一 `**教材原文**（第X页）："……"` 格式
   - 课堂补充统一 `> ⚠️ 课堂补充：` 前缀
4. **产出文件**：
   - `经济史完整复习提纲_修订版.md` — 最终可打印背诵文档
   - `CHANGELOG.md` — 完整变更日志（页码修正数、省略号修复数、内容补充数、缺口状态）

## 与 /do-agent 的关系

`/study-guide-generator` 是 `/do-agent` 的上层封装：
- `/do-agent` 提供通用多代理流水线框架（规划→执行→审阅→修订）
- `/study-guide-generator` 将框架具体化为"教材→复习资料"的专用流水线（含审阅分工模板、质量标准、输出格式规范）

当用户说 `/study-guide-generator` 时，实际上是调用本Skill的完整闭环逻辑，底层使用 `/do-agent` 的多代理编排能力。

## 已知边界

- **教材无直接论述的考点**（需标注课堂补充）：常见于诺斯悖论、斯密vs李斯特、哈耶克vs凯恩斯等西方理论，以及延期殿廷辩、金圆券改革等教材未覆盖的历史事件。审阅阶段应在全chunk搜索后确认。
- **OCR精度限制**：部分数字可能误识别（如"6%"被OCR为"69"），审阅时应标注歧义。
- **页码偏移**：PDF页码与教材印刷页码常存在+10~+12的系统性偏移，需在阶段一就统一标准。
- **相邻代理重叠**：每两个相邻审阅代理之间应有1-2个chunk的重叠覆盖，防止边界考点遗漏。

